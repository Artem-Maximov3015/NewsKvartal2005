# Telegram-бот новостей «Квартал-2005»

Бот отслеживает раздел «Новости» сайта квартал2005.рф
(`https://квартал2005.рф/novosti`) и присылает новые публикации в
Telegram-чат, группу или канал. Работает без отдельного сервера —
запускается по расписанию через **GitHub Actions**, а история уже
отправленных новостей хранится прямо в репозитории на GitHub (файл
`data/seen_news.json`), который обновляется автоматическим коммитом
после каждой проверки.

## Как это работает

1. GitHub Actions по расписанию (по умолчанию — каждые 30 минут)
   запускает `bot/main.py`.
2. Скрипт скачивает страницу `/novosti`, находит в ней новости
   (дата + заголовок + ссылка).
3. Сравнивает их со списком уже отправленных (`data/seen_news.json`).
4. Новые новости отправляет в Telegram через Bot API.
5. Обновлённый список «увиденных» новостей коммитится обратно в
   репозиторий — это и есть «база данных» бота, версионируемая Git'ом.

## Структура проекта

```
kvartal2005-news-bot/
├── .github/workflows/check_news.yml   # расписание запуска (GitHub Actions)
├── bot/
│   ├── main.py             # основная логика: сравнение, отправка, сохранение
│   ├── scraper.py          # парсер раздела новостей
│   └── telegram_sender.py  # отправка сообщений через Telegram Bot API
├── data/seen_news.json     # список уже отправленных новостей (авто-обновляется)
├── requirements.txt
├── .env.example             # пример переменных для локального запуска
└── .gitignore
```

## Шаг 1. Создать Telegram-бота

1. Откройте в Telegram **@BotFather**.
2. Отправьте `/newbot`, придумайте имя и username бота.
3. BotFather выдаст токен вида `123456789:AA...` — это `TELEGRAM_BOT_TOKEN`.

## Шаг 2. Узнать chat_id

Куда слать новости — решаете вы: личный чат с ботом, группа или канал.

- **Канал:** добавьте бота администратором канала. chat_id канала —
  это его username с `@` (например, `@kvartal2005_news`) либо числовой
  ID, который можно получить через `@username_to_id_bot` или метод
  `getChat`.
- **Группа:** добавьте бота в группу, напишите туда любое сообщение,
  затем откройте в браузере
  `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найдите поле
  `chat.id` (для групп он отрицательный, например `-1001234567890`).
- **Личный чат:** напишите боту `/start`, затем тоже посмотрите
  `getUpdates` — там будет ваш `chat.id`.

cd kvartal2005-news-bot

git branch -M main
git remote add origin https://github.com/<ваш_логин>/<репозиторий>.git

## Шаг 4. Добавить секреты в GitHub

В репозитории: **Settings → Secrets and variables → Actions → New
repository secret**. Добавьте:

| Имя секрета            | Значение                                   |
|-------------------------|---------------------------------------------|
| `TELEGRAM_BOT_TOKEN`    | токен от @BotFather                         |
| `TELEGRAM_CHAT_ID`      | chat_id из шага 2                           |
| `NEWS_URL` (необязательно) | по умолчанию уже стоит `https://квартал2005.рф/novosti` |

## Шаг 5. Включить и проверить Actions

1. Перейдите во вкладку **Actions** репозитория — если GitHub
   спросит, разрешите запуск workflow.
2. Откройте workflow **Check Kvartal2005 News** и нажмите
   **Run workflow** (это запуск вручную, не дожидаясь расписания).
3. Проверьте логи запуска — там будет видно, какие новости найдены и
   отправлены, либо текст ошибки.

После первого успешного запуска бот будет сам проверять новости каждые
30 минут (это задаётся в `.github/workflows/check_news.yml`, строка с
`cron`). Учтите: GitHub может задерживать запуск scheduled workflow на
несколько минут при высокой нагрузке на свои серверы — это нормально
и не зависит от вашего кода.

## Если парсер не находит новости

Сайт квартал2005.рф закрывает автоматический доступ к страницам в
своём `robots.txt`, поэтому я не смог напрямую открыть `/novosti` и
снять точную HTML-разметку при подготовке бота. Парсер в
`bot/scraper.py` написан по эвристике «дата `дд.мм.гггг` рядом с
заголовком/ссылкой», которая совпадает с тем, что видно в открытых
поисковых сниппетах сайта, и протестирован на похожей по структуре
странице — но если реальная верстка устроена иначе, нужно немного
подправить `scraper.py`:

1. Откройте `https://квартал2005.рф/novosti` в браузере.
2. Нажмите **F12** (или правой кнопкой → «Просмотреть код») и
   найдите в HTML блок со списком новостей.
3. Посмотрите, как там оформлены дата, заголовок и ссылка — в одном
   `<a>` или в разных тегах (`<span>`, `<div>` и т.д.).
4. В файле `bot/scraper.py` функция `fetch_news()` уже перебирает
   два сценария — при необходимости добавьте под найденную верстку
   третий блок по аналогии (нужный CSS-селектор можно получить через
   «Copy → Copy selector» в DevTools).

Буду рад помочь донастроить парсер, если пришлёте мне фрагмент HTML
со страницы новостей (например, через «Просмотреть код страницы» →
скопировать кусок с одной новостью).

## Локальный запуск (для проверки перед деплоем)

```bash
cd kvartal2005-news-bot
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env            # заполните токен и chat_id
export $(grep -v '^#' .env | xargs)   # Linux/macOS

python bot/main.py
```

## Настройка частоты проверки

Откройте `.github/workflows/check_news.yml` и измените строку cron,
например:

- `*/15 * * * *` — каждые 15 минут
- `0 * * * *` — раз в час
- `0 8,20 * * *` — два раза в день, в 08:00 и 20:00

## Важно об уважительном доступе к сайту

У сайта стоит ограничение в `robots.txt` на автоматических роботов.
Этот бот делает по одному запросу к странице за один запуск (не чаще,
чем раз в выбранный интервал), не сохраняет и не публикует ничего
кроме заголовка, даты и ссылки на новость — то есть ведёт себя как
обычный читатель, который раз в N минут обновляет страницу, а не как
массовый краулер. Тем не менее не делайте интервал слишком частым
(чаще, чем раз в 5–10 минут) — это не нужно для новостной ленты и
создаёт лишнюю нагрузку на сайт ЖКХ-компании.
